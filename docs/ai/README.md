# AI Workflow (Ticket-First)

This folder is the **canonical location** for the "plan → backlog → ticket-by-ticket" workflow.

## Start Here
1. `NON_NEGOTIABLES.md` — rules that are never violated
2. `DEFINITION_OF_DONE.md` — when is a ticket truly done
3. `TICKETS_TEMPLATE.md` — full ticket format (copy/paste)
4. `HANDOFF_RULES.md` — cross-model handoff format
5. `CONTEXT_POLICY.md` — context management rules

## Templates / Policies
- `TICKETS_TEMPLATE.md` — ticket structure for one-pass implementation
- `DEFINITION_OF_DONE.md` — unified DoD
- `NON_NEGOTIABLES.md` — quality & security guardrails
- `HANDOFF_RULES.md` — cross-model communication format
- `CONTEXT_POLICY.md` — what to read, when, how much

## Backlogs
Backlogs live outside `docs/ai/` to keep this folder focused on templates.

- Active: `docs/backlogs/active/`
