# Plan Protocol

Минимално правило: всяка работа минава през ticket (в backlog-а) с дефинирани “done” и “verify”.

## Планът живее в backlog-а

- Backlogът е реалният “план за внедряване/бъгове”.
- `Ticket Tracker` има `Next:` и това е единствената “опашка”.

## Забранено

- Работа без ticket (освен emergency, което се записва като ticket след това).
